# Реализация процесса обработки данных.
Достаточно запустить script.sh, который:
1. Подключается на удалённую машину
2. Устанавливает prefect.sh
3. Переносит на удалённую машину скрипт `prefect_process.py`
4. Запускает его.
Далее приведено описание python скрипта

### 1. `start_spark()` — Инициализация Spark-сессии
- Подключается к кластеру через **YARN**
- Указывает Hive Metastore: `thrift://tmpl-nn:9083`
- Возвращает объект `SparkSession`

### 2. `extract(spark)` — Извлечение данных
- Подключается к **HDFS** (`tmpl-nn:9000`)
- Читает файл `/raw/test.zstd.parquet` в формате **Parquet**
- Возвращает `DataFrame`

### 3. `transform(spark, df)` — Трансформация данных
- Выбирает все колонки (`SELECT *`)
- Добавляет новую колонку `dow_str` — строковое представление `date_dow`
- Возвращает коннектор `Hive` и трансформированный `DataFrame`

### 4. `load(hive, df)` — Загрузка данных
- Записывает `DataFrame` в таблицу **`default.test_prefect_partitioned`**
- Режим записи: **полная замена таблицы** (`replace_entire_table`)
- Партиционирование по колонке **`dow_str`**

### 5. `stop_spark(spark)` — Завершение Spark-сессии
- Освобождает ресурсы кластера
